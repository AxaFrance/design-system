## Modes

The Radio component has 4 modes: `default`, `classic` `cardRadio` and `inline`.
You can see them in action in the stories above.

## Orientation (Radio Card only)

When `cardRadio` mode is selected, 2 orientations are available :

- `vertical`, options are vertical cards - _default value_
- `horizontal`, options are horizontal cards

## Customize width and height (Radio Card only)

Radio cards can have specific size depending on their content or what's expected in your prototype. You can customize `width`and `height`by your own.

You need to set values on CSS properties `--width-radio-card` and/or `--height-radio-card`

```css
.custom-size-card {
  --width-radio-card: 15rem;
  --height-radio-card: 95px;
}
```

```tsx
const options = [
  { label: "Paris", value: "paris", icon: villaIcon },
  { label: "Lille", value: "lille", icon: villaIcon },
  { label: "Madrid", value: "madrid", icon: villaIcon },
];

const MyRadioCard = () => (
  <Radio mode="cardRadio" className="custom-size-card" options={options} />
);
```

## Required

The component can be required. In that case, the label will be followed by a red asterisk. In order to make the component required, set the `required` prop to `true`.

Alternatively you can to add to the `classModifier` the value `required`. This behaviour exists to keep backward compatibility but might disapear in the future.
